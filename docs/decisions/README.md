# Architecture Decision Records

Short, dated records of consequential, hard-to-reverse architecture decisions and **why** they
were made — so a decision is made once, not re-litigated every few months.

Write a new ADR when a choice (a) is expensive to reverse, (b) shapes the system's structure or
trust boundary, or (c) has been proposed-and-rejected before and keeps coming back. Number them
sequentially; never edit a decided ADR's intent — supersede it with a new one.
