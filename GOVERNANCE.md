# NAVIG Governance

## Governance Model

NAVIG uses a maintainer-led model with transparent public discussion.
Maintainers make final merge/release decisions after community review.

## Maintainers

- NAVIG Core Team (lead maintainers listed in repository settings)

## Decision Process

1. Issues for defects and focused feature requests.
2. Discussions for roadmap, architecture, and RFC-level proposals.
3. Pull requests for implementation.

Major/breaking changes should be proposed via RFC discussion first.

## Contribution Rules

- Follow `CONTRIBUTING.md`.
- Keep PRs scoped and test-backed.
- Document behavioral changes and migration impact.
- Security-sensitive changes require explicit reviewer sign-off.

## Community Channels

- GitHub Issues
- GitHub Discussions (enabled at repository settings)
- Security channel: `SECURITY.md`

If Discussions are not enabled yet, open governance topics through Issues with `discussion` label.

## Code of Conduct

See `CODE_OF_CONDUCT.md`.
